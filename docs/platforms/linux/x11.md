# X11 — Specifics and Limitations

---

## Overview

Under X11, PyAutoGUI2 uses **python-xlib** to control the mouse and keyboard. This provides
reliable, low-level access to the X server without requiring any additional system configuration
beyond the Python package itself.

---

## How It Works

| Feature | Implementation |
|---|---|
| Mouse movement | Xlib `XWarpPointer` |
| Mouse clicks | Xlib `XTestFakeButtonEvent` |
| Keyboard input | Xlib `XTestFakeKeyEvent` |
| Pointer position | Xlib `XQueryPointer` |
| Screen size | Xlib `XDisplayWidth` / `XDisplayHeight` |
| Keyboard layout | `setxkbmap -query` |

No kernel modules, no elevated privileges, and no additional system packages are required.

---

## Known Limitations

### Keyboard layout must be QWERTY

The python-xlib implementation used internally has known issues with non-QWERTY keyboard layouts
(AZERTY, QWERTZ, etc.). As a result, **only QWERTY is fully supported under X11**.

**What this means in practice:**

- If your system layout is QWERTY: everything works as expected.
- If your system layout is not QWERTY: PyAutoGUI2 will emit a warning at startup and keyboard
  functions will raise an error when called. Pointer, Screen, and Dialogs functions are unaffected
  and will work normally regardless of the active layout.

> **Warning emitted at startup (non-QWERTY layout detected):**
> ```
> WARNING: Detected keyboard layout 'AZERTY'. Only QWERTY layouts are officially supported under X11.
> ```

If your automation only uses mouse control, screen operation, or dialogs you can safely ignore this warning.

## Troubleshooting

### "python-xlib is not installed"

```bash
pip install python-xlib
```

---

### Keyboard functions raise an error

Verify your current keyboard layout:

```bash
setxkbmap -query | grep layout  # should output "us" for QWERTY
```

If your layout is not `us` (QWERTY), keyboard automation is not supported under X11. Consider
switching to a QWERTY layout, or — if your distribution supports it — running your session
under Wayland instead. See [Wayland — Specifics and Limitations](wayland.md).

---

### Script works on X11 but not inside a Wayland session (XWayland)

Some Wayland compositors provide an XWayland compatibility layer that allows X11 applications
to run on Wayland. PyAutoGUI2 may work in this context, but it is not officially supported.
If you are running a Wayland session, refer to [Wayland — Specifics and Limitations](wayland.md)
instead.
